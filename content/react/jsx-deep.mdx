## JSX is Syntax Sugar

Every JSX element compiles to a `React.createElement` call.

```tsx
// What you write:
const el = <h1 className="title">Hello</h1>

// What Babel produces:
const el = React.createElement("h1", { className: "title" }, "Hello")
```

JSX makes the tree readable. `createElement` is what the runtime actually runs.

## Expressions in JSX

Any JavaScript expression goes inside `{}`.

```tsx
function Greeting({ user, unread }) {
  const now = new Date().getHours()
  const timeGreeting = now < 12 ? "Good morning" : "Good afternoon"

  return (
    <div>
      <h1>{timeGreeting}, {user.name}</h1>
      <p>{user.name.length > 0 ? `You have ${unread} messages` : "No messages"}</p>
      <p>{Math.random().toFixed(2)}</p>
    </div>
  )
}
```

Statements (`if`, `for`, `switch`) don't fit inside `{}` — extract to a variable or use ternary.

## Conditional Rendering

Different patterns for different situations.

```tsx
// Show badge only if unread > 0
function NotificationBadge({ unread }) {
  return (
    <div>
      <p>Messages</p>
      {unread > 0 && (
        <span className="badge">{unread}</span>
      )}
    </div>
  )
}
```
⚠️ `{0 && <Component />}` renders `0`, not nothing. Compare with `{count > 0 && <C />}`.

```tsx
function StatusBanner({ status }) {
  return (
    <div className={status === "active" ? "bg-green" : "bg-gray"}>
      {status === "active" ? "Online" : "Offline"}
    </div>
  )
}
```

Ternary works inline. Good for two branches.

```tsx
function Dashboard({ role }) {
  if (role === "admin") return <AdminPanel />
  if (role === "user") return <UserPanel />

}
```

Early return with `if` keeps logic flat. Best for exclusive branches.

## Lists and Keys

Map over arrays to render lists. Every item needs a **stable key**.

```tsx
interface Todo {
  id: string
  text: string
  done: boolean
}

function TodoList({ items }: { items: Todo[] }) {
  return (
    <ul>
      {items.map((item) => (
        <li key={item.id}>
          {item.text} {item.done ? "✓" : "○"}
        </li>
      ))}
    </ul>
  )
}
```

Key requirements:
- **Stable** across re-renders — use unique IDs from your data
- Avoid `index` — it breaks when items reorder, causing stale state

<Alert>
  <Info />

## Fragments

Group elements without adding a DOM node.

```tsx
function Columns() {
  return (
    <>
      <td>First</td>
      <td>Second</td>
    </>
  )
}
```

`<>` is shorthand for `<React.Fragment>`. Keyed fragments need the full `<Fragment key={id}>`.

## Spread Props

Pass an object's properties as props with `{...obj}`.

```tsx
interface User {
  name: string
  email: string
  role: string
}

function ProfileCard({ name, email, role }: User) {
  return (
    <div className="card">
      <h2>{name}</h2>
      <p>{email}</p>
      <Badge>{role}</Badge>
    </div>
  )
}

// Spread user object
const user: User = { name: "Alice", email: "a@b.com", role: "admin" }
;<ProfileCard {...user} />
```
