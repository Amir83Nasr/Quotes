# ES Modules

<LearningObjectives objectives={["Use named and default exports/imports", "Dynamically import with import()", "Explain tree-shaking", "Compare CommonJS vs ESM"]} />

## Named Exports

```javascript
// math.js
export const PI = 3.14159
export function add(a, b) { return a + b }
// app.js
import { PI, add } from "./math.js"
import * as math from "./math.js"
import { add as sum } from "./math.js"
```

## Default Exports

One per module. No name required at import.

```javascript
// logger.js
export default function log(msg) { console.log("[LOG]", msg) }
// app.js
import log from "./logger.js"     // any name works
import User, { ROLES } from "./user.js"  // mix default + named
```

## Dynamic import()

Load on demand — code splitting.

```javascript
// Static: bundled eagerly
import { format } from "date-fns"
// Dynamic: code-split
if (isAdmin) {
  const admin = await import("./admin-panel.js")
  admin.init()
}
```

<Alert><Lightbulb /><AlertTitle>Code Splitting</AlertTitle><AlertDescription>Dynamic import() creates separate chunks loaded only when needed.</AlertDescription></Alert>

## Module Resolution

```javascript
import { useEffect } from "react"      // bare specifier → node_modules
import { helper } from "./utils.js"     // relative — extension required
```

## Tree-Shaking

Bundlers remove unused exports traced from static imports.

```javascript
// ✅ Named exports: tree-shakeable
export function used() {}
export function unused() {}  // eliminated
// ❌ export default { a, b } — hard to tree-shake
```

<Alert><AlertTriangle /><AlertTitle>Anti-Pattern</AlertTitle><AlertDescription>`export * from "./utils"` (barrel files) kills tree-shaking. Direct imports are safer.</AlertDescription></Alert>

## CommonJS vs ESM

<Tabs><TabsList><TabsTrigger value="c">CJS</TabsTrigger><TabsTrigger value="e">ESM</TabsTrigger></TabsList><TabsContent value="c">

```javascript
const { PI } = require("./math.cjs")
// Sync, runtime, no tree-shaking, no top-level await
```

</TabsContent><TabsContent value="e">

```javascript
import { PI } from "./math.js"
// Async, parse-time, tree-shaking, top-level await
```

</TabsContent></Tabs>

| Feature | CJS | ESM |
|---------|-----|-----|
| Syntax | require/exports | import/export |
| Loading | Sync, runtime | Async, parse-time |
| Tree-shaking | No | Yes |
| Top-level await | No | Yes |
